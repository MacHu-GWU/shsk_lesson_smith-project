# Record File Creation and Edits with Commits

## 1. Goal

Take one file through creation and multiple edits, leaving a clear record for every change.

---

## 2. What to Do

1. Create a new file notes.md in your repo and commit it.
2. Edit the file at least once more and commit again.
3. Open the file's History and review each commit message.

**Estimated time:** 10 to 15 minutes

---

## 3. Checklist

- [ ] **File created**: the repo shows the new notes.md.
- [ ] **At least two commits**: the file's History has at least one creation and one edit commit.
- [ ] **Messages readable**: every commit message lets someone else understand what changed.
