# 用 Commit 记录文件的创建与修改

## 1. 目标

完成一个文件从创建到多次修改的完整过程, 并让每次改动都留下清晰的记录.

---

## 2. 要做的事情

1. 在自己的 repo 里创建一个新文件 notes.md 并提交.
2. 再编辑这个文件至少一次并提交.
3. 打开文件的 History, 核对每条 commit 的说明.

**预计用时:** 10 到 15 分钟

---

## 3. 检查清单

- [ ] **文件已创建**: repo 里能看到新建的 notes.md.
- [ ] **至少两条 commit**: 文件的 History 里有创建和修改各至少一条记录.
- [ ] **Message 合格**: 每条 commit message 都能让别人看懂这次改了什么.
