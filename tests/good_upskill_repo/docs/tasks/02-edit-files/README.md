# Create and Edit Files in a Repository

> Teaches you to create files, edit them, and record every change with commits on the GitHub web UI.

## 1. Overview

With a repo in place, the next step is putting content in it. The GitHub web UI lets you create and edit files directly. Every save is a commit: a described, traceable record of the change.

---

## 2. Learning Goals

A file-sharing drive can store files too, but it cannot answer "what changed in this file last week, and why". Commit history gives every change an author, a time, and a reason. That is the essential difference between version control and bare storage.

After this task, you will be able to:

1. Create new files and edit existing ones on the GitHub web UI.
2. Write a commit message that clearly describes the change.
3. Inspect a file's past changes through its History.

---

## 3. Hands On

1. Open the repo from the previous lesson, click Add file, then Create new file.
2. Name it notes.md and write a few lines, for example what you learned today.
3. Describe the change in one sentence in the commit message, for example Add first note, then commit.
4. Back in the file list, open notes.md and click the pencil icon to enter edit mode.
5. Append one line, write another commit message describing this change, and commit.
6. Click History on the file page and confirm you can see both commits.

---

## 4. Recap

- Every save is a commit; the commit message should state what changed and why.
- A file's History is its full resume; any past version can be recovered.
- The web UI suits small edits; larger changes will later go through a local clone and push.
