# Create a Branch and Commit on It

## 1. Goal

Experience branch isolation: changes happen only on your branch while main stays unchanged.

---

## 2. What to Do

1. Create a new branch from main in your repo, named in lowercase with hyphens.
2. Edit a file on the new branch and commit at least once.
3. Switch back to main and compare the same file across the two branches.

**Estimated time:** 10 to 15 minutes

---

## 3. Checklist

- [ ] **Branch created**: the branch dropdown shows the new branch, named per the lowercase-hyphen convention.
- [ ] **Change committed**: the new branch has at least one new commit.
- [ ] **Main untouched**: you can demonstrate the same file differing between main and the new branch.
