# Build Your First Public Repository

## 1. Goal

Under your own GitHub account, use the browser to create a public repository with a README, and be able to explain two things in your own words: what a repository is, and what a commit is. This is the first step of the "create repo -> edit files -> branch -> merge" collaboration loop, and the foundation for every lesson that follows.

---

## 2. What to Do

1. Log in to github.com, click the plus sign in the top right corner, and choose New repository.
2. Fill in the Repository name using all lowercase with hyphens, for example recipe-box.
3. Set Visibility to Public.
4. Check Add a README file so GitHub makes the first commit for you.
5. Click Create repository, confirm the page jumps to the new repo's home, and check that the README content is visible.
6. Open the commit history, find the auto-generated Initial commit, and see clearly what it recorded.

**Estimated time:** 5 to 10 minutes

---

## 3. Checklist

- [ ] **Repo created**: a new repository with its own URL appears under your account.
- [ ] **Naming convention**: the repository name is all lowercase with words separated by hyphens.
- [ ] **Public visibility**: the repo was created as Public.
- [ ] **README in place**: the README content is visible directly on the repo home page.
- [ ] **First commit present**: you can find the Initial commit in the commit history and say what it recorded.
- [ ] **Can explain the concepts**: you can explain in your own words how a repository differs from a plain folder, and what a commit is.
