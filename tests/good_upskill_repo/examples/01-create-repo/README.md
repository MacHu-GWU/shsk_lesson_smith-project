# Create Your First GitHub Repository

> Teaches you to create a well-configured GitHub repository from scratch.

## 1. Overview

A repository (repo for short) is a project's home on GitHub: it stores files, records every change, and supports collaboration and access control. Creating a repo is step one of every digital project.

---

## 2. Learning Goals

Without a repo, teamwork degrades into files flying around in email and chat: nobody can tell who changed what or which version is current. Creating a repo gives the project a single, traceable home.

After this task, you will be able to:

1. Explain what a repository is and how it differs from a plain folder.
2. Independently create a well-configured repository.
3. Explain what the visibility, README, and .gitignore settings do.

---

## 3. Hands On

1. Sign in to GitHub, click the plus icon in the top-right corner, and choose New repository.
2. Fill in the repository name: all lowercase, words separated by hyphens, for example my-first-project.
3. Set visibility to Private (the default choice for work projects; you can switch to Public later).
4. Turn on Add a README so the repo introduces itself from day one.
5. Pick a .gitignore template matching your project type, or None if unsure.
6. Click Create repository and confirm you land on the new repo's homepage.

---

## 4. Recap

- A repository is the project container, the unit of collaboration, and the smallest unit of access control.
- Name with lowercase and hyphens; default visibility to Private; always enable the README.
- Make creating a repo the first step of every new project.
