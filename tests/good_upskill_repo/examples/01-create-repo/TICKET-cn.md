# 建出你的第一个公开 Repository

## 1. 目标

在你自己的 GitHub 账号下, 全程用浏览器建出一个带 README 的 public repository, 并能用自己的话讲清楚两件事: repository 是什么, commit 是什么. 这是走完 "建仓库 -> 改文件 -> 开分支 -> 合并" 这条协作闭环的第一步, 也是后面所有课程的地基.

---

## 2. 要做的事情

1. 登录 github.com, 点右上角的加号, 选 New repository.
2. 填 Repository name, 用全小写加连字符, 例如 recipe-box.
3. Visibility 选 Public.
4. 勾上 Add a README file, 让 GitHub 自动帮你完成第一个 commit.
5. 点 Create repository, 确认跳转到新 repo 的主页, 页面上能看到 README 内容.
6. 点进 commit 历史, 找到那条自动生成的 Initial commit, 看清它记录了什么.

**预计用时:** 5 到 10 分钟

---

## 3. 检查清单

- [ ] **Repo 建成**: 你的账号下出现了一个新的 repository, 有独立网址.
- [ ] **命名规范**: repository 名字全小写, 用连字符分隔单词.
- [ ] **Public 可见性**: repo 创建为 Public.
- [ ] **README 就位**: repo 主页能直接看到 README 的内容.
- [ ] **第一个 commit 在场**: 能在 commit 历史里找到 Initial commit, 并说清它记了什么.
- [ ] **能讲清概念**: 能用自己的话解释 repository 与普通文件夹的区别, 以及 commit 是什么.
