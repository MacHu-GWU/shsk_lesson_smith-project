# Land Your First Few Commits with the Web Editor

## 1. Goal

Without installing git at all, use GitHub's web editor to create and change files in the repository you already have, giving every change a commit message that states its intent. When you are done, your Commits page should show a string of history that reads clearly.

---

## 2. What to Do

1. Open the repository you created in Task 01 (for example recipe-box).
2. Use Add file, Create new file to add a file whose name includes a folder, such as recipes/pancakes.md, and write a few lines of Markdown.
3. When committing, write a commit message that starts with a verb and reads as one sentence, for example Add pancake recipe.
4. Open README.md, edit it with the pencil icon, add a line introducing the project, and commit again with a message such as Update README with project intro.
5. Go to the Commits page, find the commits you just made, and click one open to view its diff (green for additions, red for deletions).

**Estimated time:** 10 to 15 minutes

---

## 3. Checklist

- [ ] **New file created**: a new file appears in the repository, and the slash in its name auto-created the matching folder.
- [ ] **File edited**: an existing file (such as README.md) was changed and committed.
- [ ] **Clear commit messages**: each message starts with a verb and states in one sentence what changed, with no vague words like update or fix.
- [ ] **Reads the history**: you can find your commits on the Commits page and understand a commit's diff when you open it.
