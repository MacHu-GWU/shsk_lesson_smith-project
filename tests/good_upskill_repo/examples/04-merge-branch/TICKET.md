# Complete a Merge with a Pull Request

## 1. Goal

Walk through a full PR cycle: open it, review the diff, merge, and clean up the branch.

---

## 2. What to Do

1. Open a Pull Request for the branch from the previous lesson, targeting main.
2. Review the diff under Files changed and confirm the changes look right.
3. Merge the PR, delete the branch, and verify the change landed on main.

**Estimated time:** 10 to 15 minutes

---

## 3. Checklist

- [ ] **PR created**: the PR has a title and a one-line summary, going from the branch into main.
- [ ] **Diff reviewed**: you can state exactly what this PR changes.
- [ ] **Merged and cleaned up**: the PR is merged, the branch is deleted, and main shows the change.
