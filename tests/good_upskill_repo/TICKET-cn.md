# 完成 GitHub 基础协作全部课程

## 1. 目标

走完本课程全部 mini task, 建立起对 GitHub 协作最小闭环的完整体感: 建仓库, 改文件, 开分支, 用 Pull Request 合并回主干. 学完你应该能不看教程, 独立在自己的账号上把这条闭环从头走一遍.

---

## 2. 要做的事情

1. 先读根目录的 README, 明白这门课怎么学.
2. 读 examples 的系列索引, 对整门课有张地图.
3. 按编号顺序刷完 examples 下全部 mini task, 每个都真在自己的 GitHub 账号上操作一遍, 并照它自己的 TICKET 逐项确认.
4. 全部刷完后, 用 upskill-quiz 自测, 测到 70% 的题都能思路清晰地答出来.

**预计用时:** 2 到 3 小时

---

## 3. 检查清单

- [ ] **全部 mini task 完成**: examples 下每个 task 的 TICKET 检查清单都已逐项打勾.
- [ ] **自测达标**: 用 upskill-quiz 测到 70% 的题都能思路清晰地答出来.
- [ ] **能独立复现**: 不看教程, 能从建 repository 一路做到用 Pull Request 合并.
- [ ] **能讲清概念**: 能用自己的话解释 repository, commit, branch, Pull Request, merge 各自解决什么问题.

---

## 4. 关键能力

- 能在 GitHub 上从零创建一个公开 repository, 并初始化 README.
- 能说清 repository 与 commit 各自是什么, 以及它们的关系.
- 能用 web editor 新建和修改文件, 并为每次改动写清晰的 commit message.
- 能读懂一个 repository 的 commit history, 看出每次改了什么.
- 能建一个 branch, 在它上面改动而不影响主干.
- 能开一个 Pull Request, 说清 code review 在团队协作里的作用.
- 能把 Pull Request 合并回默认分支, 完整走完一次协作闭环.
- 遇到 merge conflict 时能认出它是什么 (不要求会解决).
