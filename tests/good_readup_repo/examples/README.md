# GitHub Collaboration Basics Series Index

This course breaks one core GitHub collaboration into five mini tasks. The first four walk you through the full loop in order (create repo, edit files, branch, merge); the last one helps you cement what you learned. Work from 01 straight through by number, and actually do each one on your own GitHub account.

## 1. Set up your repository

First, settle where things live and how changes get recorded. This group gets you a real repository and teaches you to record every change with a commit.

- [01-create-repo](01-create-repo/README.md): Create your first repository from scratch and understand what a repository and a commit are.
- [02-edit-files](02-edit-files/README.md): Create and edit files in the browser, recording each change with a clear commit message.

---

## 2. Branches and Pull Requests

This group is the heart of collaboration: learn not to edit the main line directly, but to open a branch to work on, then merge it back through a reviewed Pull Request.

- [03-git-branch](03-git-branch/README.md): Use a branch to isolate unfinished work from the main line, and understand why collaboration depends on branches.
- [04-merge-branch](04-merge-branch/README.md): Merge a branch back into the main line through a reviewed Pull Request, completing the whole loop.

---

## 3. Level up

After you have learned the content, this group helps you confirm you truly get it and points you to what is next.

- [05-level-up](05-level-up/README.md): Review the whole collaboration loop and get directions and search keywords for going further.

---

## 4. Wrap-up

Once you finish these five mini tasks, you fully command the most common collaboration loop on GitHub: from creating a repo, to recording changes with commits, to isolating work on a branch, to reviewing and merging through a Pull Request. This loop is the foundation for later learning the git command line, team branching strategies, and automation.
