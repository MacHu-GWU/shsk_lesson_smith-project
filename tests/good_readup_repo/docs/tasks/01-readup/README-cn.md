# GitHub 基础协作课程总览

> 一句话: 这门课在浏览器里带你走通 GitHub 的最小协作闭环. 想先了解全貌, 看 [README-ORIGINAL-cn.md](README-ORIGINAL-cn.md).

## 1. 这门课是什么

这门课教你完全用浏览器完成 GitHub 上的一次完整协作: 建 repository, 用 commit 改文件, 开 branch, 再用 Pull Request 合并回主干. 面向完全没用过 GitHub 的新手, 不需要装 git, 不需要命令行.

这是一个 readup 类型的教学仓库: 整个 repo 就是一门课, 内容拆成 [examples/](examples/) 下一个个按编号排列的 mini task. 你不用把它当成从头读到尾的教材, 而是照下面 "怎么读" 的路子, 一篇篇顺着读下来, 边读边照每篇的 TICKET 动手.

---

## 2. 怎么读

1. 先读 [examples/README-cn.md](examples/README-cn.md) 的系列索引, 对整门课有张地图.
2. 按编号顺序读 examples 下的 mini task (01, 02, ...): 每篇先读 README, 再照它的 TICKET 真在自己的 GitHub 账号上做一遍, 把检查清单逐项打勾再往下.
3. 一篇读完再读下一篇, 直到读完最后一篇 (它会帮你梳理学到了什么, 并指出想更进一步往哪走).
4. 读到什么算成: 每个 mini task 的 TICKET 检查清单都能逐项打勾, 且能不看教程自己把整条闭环走一遍.

---

## 3. 课程内容

全部 mini task 按主题梳理在 [examples/README-cn.md](examples/README-cn.md) 里, 从那儿进入.

---

## 4. 读完你能做到什么

你能独立在 GitHub 上从零建起一个项目, 用清晰的 commit 记录改动, 用 branch 隔离未完成的工作, 再用 Pull Request 把改动 review 后合并回主干, 完整走通一次团队协作里最常见的流程.
