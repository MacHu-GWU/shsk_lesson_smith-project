# GitHub Collaboration Basics Course Overview

> In one line: this course walks you through GitHub's minimal collaboration loop, all in the browser. For the full pitch, see [README-ORIGINAL.md](README-ORIGINAL.md).

## 1. What this course is

This course teaches you to complete one full collaboration on GitHub entirely in the browser: create a repository, change files with commits, open a branch, and merge it back into the main line through a Pull Request. It is aimed at complete beginners who have never used GitHub. No git install, no command line.

This is a readup-style teaching repo: the whole repo is one course, and the content is split into numbered mini tasks under [examples/](examples/). You do not read it cover to cover; you follow the "How to read" path below, going through the tasks in order and doing each one yourself as you go.

---

## 2. How to read

1. Start with the series index in [examples/README.md](examples/README.md) to get a map of the whole course.
2. Work through the mini tasks under examples in order (01, 02, ...). For each one, first read its README, then do it on your own GitHub account against its TICKET, checking off the TICKET's checklist before moving on.
3. Read one task, then the next, until you reach the last one (it reviews what you learned and points you where to go further).
4. What counts as done: every mini task's TICKET checklist is fully ticked off, and you can walk the whole loop from scratch without the tutorials.

---

## 3. Course content

All the mini tasks are organized by theme in [examples/README.md](examples/README.md). Enter from there.

---

## 4. What you can do afterward

You can independently spin up a project on GitHub from scratch, record changes with clean commits, isolate unfinished work on a branch, and merge it back into the main line through a reviewed Pull Request: the full loop of the most common team collaboration flow.
