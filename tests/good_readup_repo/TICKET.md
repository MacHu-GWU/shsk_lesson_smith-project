# Finish the Whole GitHub Collaboration Basics Course

## 1. Objective

Read through every mini task in this course and build a complete feel for GitHub's minimal collaboration loop: create a repo, edit files, branch, and merge back into the main line through a Pull Request. By the end you should be able to walk this loop from scratch on your own account without the tutorials.

---

## 2. What to do

1. Read the root README first so you know how to read this course.
2. Read the examples series index to get a map of the whole course.
3. Read every mini task under examples in order, actually doing each one on your own GitHub account and confirming it against that task's own TICKET.
4. Read the review-and-level-up task last, and settle what you learned and where to go next.

**Estimated time:** 2 to 3 hours

---

## 3. Checklist

- [ ] **All mini tasks done**: every task under examples has its TICKET checklist fully ticked off.
- [ ] **Can reproduce solo**: without the tutorials, you can go from creating a repository all the way to merging via a Pull Request.
- [ ] **Can explain the concepts**: you can say in your own words what repository, commit, branch, Pull Request, and merge each solve.

---

## 4. Key abilities

- Create a public repository from scratch on GitHub and initialize it with a README.
- Explain what a repository and a commit each are, and how they relate.
- Create and edit files with the web editor, writing a clear commit message for each change.
- Read a repository's commit history and see what each change did.
- Create a branch and make changes on it without affecting the main line.
- Open a Pull Request and explain what code review does for team collaboration.
- Merge a Pull Request back into the default branch, completing a full collaboration loop.
- Recognize a merge conflict for what it is when you hit one (resolving it is not required).
