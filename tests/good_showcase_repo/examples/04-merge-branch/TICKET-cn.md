# 用 Pull Request 完成一次合并

## 1. 目标

走完一次完整的 PR 流程: 发起, 审查 diff, 合并, 清理 branch.

---

## 2. 要做的事情

1. 为上一课的 branch 发起一个 Pull Request, 目标是 main.
2. 在 Files changed 里检查 diff, 确认改动符合预期.
3. 完成 merge, 删除该 branch, 并在 main 上验证改动已生效.

**预计用时:** 10 到 15 分钟

---

## 3. 检查清单

- [ ] **PR 已创建**: PR 有标题和一句话说明, 方向是 branch 合入 main.
- [ ] **Diff 已检查**: 能说出这个 PR 具体改了什么.
- [ ] **合并且已清理**: PR 已 merge, branch 已删除, main 上能看到改动.
