# GitHub 基础协作 系列索引

这门课把 GitHub 上最核心的一次协作拆成六个 mini task. 前四个按顺序带你走一遍完整闭环 (建仓库, 改文件, 开分支, 合并), 后两个帮你检验并把它讲成作品. 建议从 01 一路按编号做下去, 每个都真在自己的 GitHub 账号上操作一遍.

## 1. 建立你的仓库

先把 "东西放哪, 改动怎么记" 搞定. 这一组让你拥有一个真正的 repository, 并学会用 commit 记录每一次改动.

- [01-create-repo](01-create-repo/README-cn.md): 从零建出你的第一个 repository, 搞懂 repository 与 commit 是什么.
- [02-edit-files](02-edit-files/README-cn.md): 在网页上新建和修改文件, 用清晰的 commit message 把每次改动记下来.

---

## 2. 分支与 Pull Request

这一组是协作的核心: 学会不在主干上直接改, 而是开一条 branch 做事, 做完再用 Pull Request 请人 review 后合并回去.

- [03-git-branch](03-git-branch/README-cn.md): 用 branch 把没写完的改动和主干隔开, 理解为什么协作离不开分支.
- [04-merge-branch](04-merge-branch/README-cn.md): 用 Pull Request 把 branch 的改动 review 后合并回主干, 走完整条闭环.

---

## 3. 检验与展示

内容学完后, 这一组帮你确认自己是真懂了, 并把它变成能讲出来的作品.

- [05-prove-i-get-it](05-prove-i-get-it/README-cn.md): 一套面试式自测题, 检验你对整门课是否知其然知其所以然.
- [06-how-i-build-this](06-how-i-build-this/README-cn.md): 讲故事底稿, 让你能把这段学习经历当作自己的作品讲给任何人听.

---

## 4. 小结

走完这六个 mini task, 你就完整掌握了 GitHub 上最常用的那条协作闭环: 从建仓库, 到用 commit 记录改动, 到用 branch 隔离工作, 再到用 Pull Request review 并合并. 而且你能讲出自己是怎么走到这一步的, 这正是把一门刷完的教程变成作品的关键.
