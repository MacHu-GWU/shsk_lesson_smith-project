# GitHub Collaboration Basics Series Index

This course breaks one core GitHub collaboration into six mini tasks. The first four walk you through the full loop in order (create repo, edit files, branch, merge); the last two help you prove you get it and turn it into a story you can tell. Work from 01 straight through by number, and actually do each one on your own GitHub account.

## 1. Set up your repository

First, settle where things live and how changes get recorded. This group gets you a real repository and teaches you to record every change with a commit.

- [01-create-repo](01-create-repo/README.md): Create your first repository from scratch and understand what a repository and a commit are.
- [02-edit-files](02-edit-files/README.md): Create and edit files in the browser, recording each change with a clear commit message.

---

## 2. Branches and Pull Requests

This group is the heart of collaboration: learn not to edit the main line directly, but to open a branch to work on, then merge it back through a reviewed Pull Request.

- [03-git-branch](03-git-branch/README.md): Use a branch to isolate unfinished work from the main line, and understand why collaboration depends on branches.
- [04-merge-branch](04-merge-branch/README.md): Merge a branch back into the main line through a reviewed Pull Request, completing the whole loop.

---

## 3. Prove it and tell it

After you have learned the content, this group helps you confirm you truly get it and turn it into your own story.

- [05-prove-i-get-it](05-prove-i-get-it/README.md): An interview-style self-test that checks whether you understand the course at the know-what and know-why level.
- [06-how-i-build-this](06-how-i-build-this/README.md): The story script for how you built this, so you can walk anyone through it as your own work.

---

## 4. Wrap-up

Once you finish these six mini tasks, you fully command the most common collaboration loop on GitHub: from creating a repo, to recording changes with commits, to isolating work on a branch, to reviewing and merging through a Pull Request. And you can tell the story of how you got there, which is what turns a finished tutorial into a portfolio piece.
