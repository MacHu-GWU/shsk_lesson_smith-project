# 运行 /lesson-smith-readup-author 的输入

> 用法: 把下面每个 "无" 换成你的想法 (还没想清楚的就留 "无"), 然后把整份连同 /lesson-smith-readup-author 一起发出去. author 会据此判断你在创作流程的哪一步, 从那一步开始引领你把 examples 下的课程内容做出来. 这里填的都是初稿, 不是死规定, 后面会边写边改.

请按 /lesson-smith-readup-author 中的规范, 引领我从零创作这一门 readup 课程.

## 1. 主题与背景

> 这门课教什么? 你为什么想做它 (动机, 契机, 想解决谁的什么问题)? 一两段话即可.

无

## 2. Scope

> 边界的初稿: 这门课多大, 多广, 多深? 哪些内容是必须要的, 哪些是绝对不要的 (点出不要的往往比点出要的更有用)? 定位在哪一档 (basic / advanced / hardcore)?

无

## 3. 读完能得到什么

> 读完这门课, 学员能做到什么, 或能回答哪些原来答不上的问题? 这会长成 README-ORIGINAL 的承诺.

无

## 4. examples 之外的 "做的部分"

> 这门课除了 examples 里的教程, 是否还有学员要读或跑的代码, 文档, 例子? 有的话简述有哪些, 以及是否已经写好 (按工作流, 这些要先完成). 纯靠 examples 讲清的课写 "无".

无

## 5. 其他偏好

> 对创作过程的额外说明: 语气口径, 深浅取向, 想参考的风格, 已经想好的 mini task 分组或篇目等. 没有写 "无".
