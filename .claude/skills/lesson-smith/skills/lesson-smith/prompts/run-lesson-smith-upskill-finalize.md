# 运行 /lesson-smith-upskill-finalize 的输入

> 用法: 把下面每个 "无" 换成你的内容 (没有就留 "无"), 然后把整份连同 /lesson-smith-upskill-finalize 一起发出去. finalize 会据此写出根目录的 README 与 TICKET (全部语种一次产出), 再跑 sync 生成 SYLLABUS 与快照, 最后跑 lint 收尾.

> 前置: 这是创作流的最后一步. 默认 examples 已全部写完, 且 /lesson-smith-upskill-forge 已产出 docs/upskill/ 与两个子 skill. 根 README 与 TICKET 结构固定, 大部分内容 finalize 会从 README-ORIGINAL, examples/README 以及各 mini task 的 TICKET 自动推导, 你只需补下面几处机器猜不准的.

请按 /lesson-smith-upskill-finalize 中的规范, 写出根目录 README 与 TICKET 的全部语种版本, 生成 SYLLABUS, 并跑 lint 收尾.

## 1. TICKET 能力清单的取舍

> 根 TICKET 末尾有一块 "关键能力" bullet (10 条以内, 需要取舍, 不是每个 mini task 都值得列, 各 mini task 自己的 TICKET 里已经说过). 你希望重点突出哪几块能力, 想砍掉哪些偏琐碎的? 没有特别要求就写 "无", finalize 自行按重要性取舍.

无

## 2. 整门课预计用时

> 从头刷完全部 mini task 大致要多久 (机器猜不准). 例如 "3 到 4 小时" 或 "一个周末". 不确定写 "无", finalize 会给个保守估计或省略.

无

## 3. 其他要求

> 对根 README 与 TICKET 的额外指示: 语气口径, 顶部那句 "承诺" (学完你能做到什么) 想怎么定, 想强调或想回避的点等. 没有写 "无".
