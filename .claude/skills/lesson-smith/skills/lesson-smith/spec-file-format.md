# Spec 文件格式约定

这个文件只在编写 lesson-smith skill 本身时参考. SKILL 运行时不引用它.

lesson-smith 的每个特殊文件的 spec (例如 `readme-spec.md`, `syllabus-spec.md`) 都按同一套格式写.

## 1. 文件即模板

spec 文件的正文, 就是这个特殊文件可以直接套用的模板: 真实的 markdown 结构, 占位内容用 `[方括号]` 标注. 创作者复制正文, 填空即可, 不用再去另一个文件对照模板.

---

## 2. 规范写在顶部注释里

所有规范性文字 (定性, 适用范围, 写作原则, 结构说明, 字段约束等) 全部放在文件最顶部的一个 `<!-- ... -->` 多行注释里. 注释之外只留模板正文.

注意: 注释内部不要出现连续两个及以上的连字符 (那会破坏 HTML 注释), 需要提到分隔线时用文字描述.

---

## 3. 注释在产出时删掉

顶部注释只服务于创作. 产出真正的文件时, 把整段注释删掉, 只留正文. 注释里要显式写明这一点. 对于脚本自动生成的文件 (例如 SYLLABUS), 脚本本来就不会带上注释, 这条自然满足.

---

## 4. 为什么这样做

模板和规范放在一起, 创作时贴身可查; 正文本身即可直接使用, 省掉 "spec 一个文件, template 又一个文件" 的来回对照, 也避免两者漂移.
