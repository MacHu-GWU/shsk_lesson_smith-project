# 我是怎么做出这个的

> 这不是一篇教程, 而是把整个 repo 打包成一个可以对别人讲的故事. 别人问你 "你会不会 XX" 或 "你怎么做的 XX" 时, 你就照这份底稿, 打开这个 repo 讲一遍你是怎么学会并做出来的. 先照默认主线排练, 再按听众临场裁剪.

## 1. 一句话故事

> [2 到 3 句的电梯版: 我用 AI 辅助, 在很短时间内把 [某技能] 从零学到能上手, 这个 repo 就是那趟学习的完整记录和产物. 只给你一句话时间就讲这个.]

---

## 2. 默认故事主线

下面七幕是这个 repo 的主线, 照顺序讲就是一条完整的故事. 每一幕先看 "怎么讲", 再看它对应 repo 里的哪个产物.

### 第 1 幕: 发现值得学

**怎么讲**

> [我在 [某场景] 里发现 [某技能] 在 [某领域] 很重要, 值得专门投资学会. 讲清这个判断是怎么来的.]

**对应产物**

> 见 [README-ORIGINAL-cn.md](../../README-ORIGINAL-cn.md) 里讲这门课定位与价值的部分.

### 第 2 幕: 让 AI 调研权威资料

**怎么讲**

> [我先让 AI 去调研 [某技能] 有哪些权威资料, 而不是自己瞎搜. 讲清怎么让 AI 收敛到靠谱来源.]

**对应产物**

> 见 [某处记录调研来源的地方] (如果 repo 里留了调研笔记就链过去; 没有就口头说).

### 第 3 幕: 蒸馏出超级专家 skill

**怎么讲**

> [我把这些资料蒸馏成一个囊括 [某技能] 的超级专家 AI agent skill, 让后面的学习都由这个专家带.]

**对应产物**

> 见 [某个 skill 或蒸馏产物的位置].

### 第 4 幕: 让专家设计一周大纲

**怎么讲**

> [我告诉这个专家我的背景和时间, 让它给我设计一个一周能执行完的课程大纲.]

**对应产物**

> 见 [examples/README-cn.md](../../examples/README-cn.md) 的系列索引, 那就是这份大纲.

### 第 5 幕: 逐点生成可练习的例子

**怎么讲**

> [大纲每一个点, 我都让 AI 出一个既能读又能上手练的例子.]

**对应产物**

> 见 examples 下按编号排列的各个 mini task (从 [01-title](../../examples/01-title/README-cn.md) 开始).

**怎么讲**

## 3. 常见追问与怎么接

这些是听众可能追问的元问题 (关于这段经历和方法, 不是考具体技术; 技术细节归 quiz). 排练时准备好怎么接.

**为什么选这个技能, 而不是别的**

> [讲清你的取舍逻辑: 为什么它值得这一周, 它解决你什么问题.]

**这些都是 AI 生成的, 你真的懂吗**

> [用能力说话: 我能不看教程独立复现, 能答自测题, 能讲清每一步为什么这么做. AI 是加速器, 不是替我理解.]

**过程中最大的坑是什么, 怎么解决的**

> [挑一个真实的卡点, 讲你怎么定位, 怎么解决的. 这段最能体现真本事.]

**这套方法能复用到别的技能吗**

> [讲清这七幕是一套可复用的学习方法, 这个 repo 只是其中一个应用.]
